One universal concept: Stages & Frames

layerJS introduces the Stage - Frame concept allowing you to handle most common UX schemes simply by marking DIVs in your HTML as Stages or Frames. 

learn more...

All elements of your site or app are placed into frames. These can contain sub pages, menus, parallax backgrounds, light-boxes and other blocks of your site. 

Stages are containers into which frames can be dynamically positioned using various transition types. Stages can contain layers which allow stacking frames on top of each other, allowing effects like floating menus and parallax.

The structure and interactivity of your site is completely defined in HTML by marking up different blocks as frames or stages. To dynamically switch active frames in your stages API calls or simply HTML links can be used. No coding required. The layout of your UI is defined by the HTML and CSS as you now it.

Just markup your HTML

Stages and Frames and all interactions are simply defined by adding attributes to your HTML.

Sample Code


<div lj-type="stage">
  <div id="layer1" lj-type="layer" lj-default-frame="main">
    <div lj-type="frame" lj-name="main" lj-fit-to="width"> 
      … your HTML code for frame 'main'… 
    </div>
    <div lj-type="frame" lj-name="second"> 
      … your HTML code for frame 'second'… 
    </div>
  </div>
</div>
